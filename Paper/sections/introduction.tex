\section{Introduction}

Retrieval-augmented generation (RAG) pipelines are sensitive to implementation choices such as chunk size, overlap, embedding model, retrieval strategy, reranking, prompt design, and answer model. Small configuration changes can affect factuality, faithfulness, latency, and cost. Systematic benchmarking is therefore needed to compare candidate pipelines under controlled conditions.

This paper describes a benchmark framework for evaluating RAG pipeline variants. The framework supports repeatable runs, structured configuration logging, metric computation, and report generation. The goal is to make design choices visible and comparable rather than relying on anecdotal model outputs.

The main contributions are:
\begin{itemize}
    \item a configurable benchmark process for RAG experiments;
    \item metric reporting for retrieval and answer generation quality;
    \item reproducible experiment artifacts suitable for comparison and analysis.
\end{itemize}
