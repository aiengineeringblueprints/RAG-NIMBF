# CHAPTER_09 Analysis: What to Incorporate & Why Scores Look High

## Source
CHAPTER_09/CHAPTER9-1_RAGA_EVAL.ipynb — RAGAS evaluation on Google's 2023 Environmental Report using GPT-4o-mini/4o and OpenAI embeddings.

## Why Scores Are Misleadingly High

CHAPTER_09 reports 93-97% on faithfulness and answer relevancy. These are not comparable to our framework's scores for three reasons:

### 1. Trivially Easy Task
- **10 synthetic single-hop questions** from a **single 100-page PDF**
- All questions generated by `SingleHopSpecificQuerySynthesizer` — each answer lives in one chunk
- No multi-hop reasoning, no cross-document retrieval, no ambiguity
- Our framework tests against harder datasets (SQuAD, t2-ragbench) with real-world diversity

### 2. GPT-4o as Judge
- Generator: GPT-4o-mini | Critic: GPT-4o | Embeddings: text-embedding-ada-002
- Our framework uses Qwen3-32B via Ollama — a much weaker model for both generation and evaluation
- OpenAI embeddings are more mature than open-source alternatives for semantic similarity scoring

### 3. Relevance Gating Inflates Scores
- A conditional answer mechanism scores retrieved context on a 1-5 scale
- If relevance < 4, the answer becomes "I don't know" (sample #9 in hybrid run)
- This pre-filters out hard cases, artificially boosting faithfulness/relevancy on the remaining samples

### 4. Low Factual Correctness Reveals the Truth
- Despite 93%+ faithfulness, **factual correctness is only 21-24%**
- The model sounds convincing (faithful to context) but doesn't actually match ground truth
- Our framework's BERTScore (0.86-0.94) and ROUGE-L (0.15-0.60) are more honest indicators

## What to Incorporate

### Worth Adopting

| Feature | What it does | Why it helps |
|---------|-------------|-------------|
| Hybrid retrieval (Dense + BM25) | Ensemble of vector search + keyword search via `EnsembleRetriever` | Our runs only test similarity/MMR. BM25 catches exact keyword matches that dense vectors miss |
| Relevance gating | LLM-based relevance check (1-5 scale) before answering | Prevents hallucinations from irrelevant context; adds a "refusal rate" metric |
| Synthetic testset generation | RAGAS `TestsetGenerator` creates Q&A pairs from any corpus | Quick smoke-testing on new documents without needing labeled data; supports persona-based query styles (misspelled, poor grammar, web-search-like) |

### Already Covered by Our Framework

- RAGAS metrics (faithfulness, answer relevancy, context precision/recall)
- Multiple retrieval strategies (similarity, MMR)
- Comprehensive metric suite (ROUGE-L, BLEU, METEOR, BERTScore, Hit@k, nDCG)

### Skip

- OpenAI model/embedding swap — our framework targets local/open-source models, which is a different (and more realistic for many deployments) evaluation scenario
